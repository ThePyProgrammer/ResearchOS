In frontend/src/components/layout/Sidebar.jsx:

3. In the ProjectsTree component (line ~716), refactor the useEffect to extract the fetch logic into a named function, then add a second useEffect that listens for the CustomEvent:

   ```javascript
   const fetchProjects = useCallback(() => {
     if (!activeLibraryId) return
     projectsApi.list({ library_id: activeLibraryId })
       .then(data => setProjects(data))
       .catch(err => console.error('ProjectsTree: failed to fetch projects:', err))
   }, [activeLibraryId])

   useEffect(() => {
     fetchProjects()
   }, [fetchProjects])

   useEffect(() => {
     const handler = () => fetchProjects()
     window.addEventListener('researchos:projects-changed', handler)
     return () => window.removeEventListener('researchos:projects-changed', handler)
   }, [fetchProjects])
   ```

   Add `useCallback` to the existing imports from 'react' at the top of the file if not already present.

   This approach uses a stable callback ref so the event listener is always pointing to the latest fetch function without stale closures.

**Graph View Integration:**

1. Add graph view toggle button in the toolbar/header area (same placement as LibraryNotes — a "Graph" button that toggles between editor view and graph view)
2. Import `NoteGraphView` from `../components/NoteGraphView`
3. Build `sourceKeyCollections` for experiment hull grouping:
   ```js
   const sourceKeyCollections = useMemo(() => {
     const map = {}
     for (const exp of experiments) {
       const groupKey = exp.parentId || exp.id
       map[`experiment:${exp.id}`] = [groupKey]
     }
     return map
   }, [experiments])
   ```
4. Build `collections` array for hull labels/colors — one entry per unique parent group:
   ```js
   const graphCollections = useMemo(() => {
     const seen = new Set()
     return experiments
       .map(exp => ({ id: exp.parentId || exp.id, name: experiments.find(e => e.id === (exp.parentId || exp.id))?.name || exp.name }))
       .filter(c => { if (seen.has(c.id)) return false; seen.add(c.id); return true })
   }, [experiments])
   ```
5. Pass to NoteGraphView:
   - `allNotes={allLoadedNotes}`
   - `collections={graphCollections}`
   - `sourceKeyCollections={sourceKeyCollections}`
   - `onNoteClick={(noteId) => { /* open note in tab */ }}`
   - `customSourceColors={{ project: '#64748b', experiment: '#f59e0b' }}`
   - `customSourceLabels={{ project: 'Project', experiment: 'Experiment' }}`
   - `storagePrefix="researchos.project-graph."` (prevents key collision with library graph per research pitfall 3)
6. Graph view renders in place of the editor area when toggle is active (same pattern as LibraryNotes)

17. Chart data protocol: copilot backend returns HTML with `<div data-chart='{"type":"bar","data":[{"name":"exp1","value":0.92}]}'>` elements

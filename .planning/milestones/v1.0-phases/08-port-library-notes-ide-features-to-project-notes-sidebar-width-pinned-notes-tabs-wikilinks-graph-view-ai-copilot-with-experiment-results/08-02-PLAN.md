  <action>
Create `frontend/src/pages/ProjectNotesIDE.jsx` by adapting the LibraryNotes.jsx patterns. This is a large component (~1500-2000 lines). Key sections:

**Data loading (top of component):**
- Receive `project`, `experiments` (flat array), `linkedPapers` from `useOutletContext()` — check what ProjectDetail passes via Outlet context
- Load project notes via `notesApi.listForProject(project.id)` on mount
- Load experiment notes for each experiment: iterate `experiments`, call `notesApi.listForExperiment(exp.id)` — store in `expNotes` state object keyed by `experiment:{expId}`
- Build `allLoadedNotes` array: project notes get `{ source: 'project', sourceName: 'Project Notes', sourceKey: 'project' }`, experiment notes get `{ source: 'experiment', sourceName: exp.name, sourceKey: 'experiment:{expId}' }`

**Sidebar (w-64, left panel):**
- Copy `ItemFolder`, `NoteRow`, `buildPath`, `ToolBtn` patterns from LibraryNotes (these are internal functions, not exported)
- "Pinned" section at top: show notes where `is_pinned === true` from `allLoadedNotes`, each as a NoteRow. Star icon toggles pin via `notesApi.update(noteId, { isPinned: !current })`
- "Recent" section below Pinned: last 8 notes by `updated_at` descending, shown as NoteRows
- "Project Notes" folder: collapsible, shows project-level notes tree
- Per-experiment folders: one `ItemFolder` per experiment from `experiments` array, collapsible, showing that experiment's notes. Use experiment name as folder label, status badge icon. Also show linked papers/websites from `experimentsApi.listPapers(exp.id)` as resource entries (PDF/website tabs)

**Tab bar (top of editor area):**
- `openTabs` state array: `{ noteId, tabType: 'note'|'pdf'|'website'|'suggestion', source, sourceKey, name }`
- `activeTabId` state: ID of active tab
- Click a NoteRow in sidebar opens that note in a tab
- Tab label: for experiment notes, prefix with truncated experiment name: `"ExpName > NoteName"` (truncate exp name to 15 chars with ellipsis)
- Close button (X) on each tab. Close removes from openTabs, selects adjacent tab
- Max tabs: 12. If exceeded, close oldest non-pinned tab

**Editor area:**
- Tiptap editor instance with all extensions from LibraryNotes: StarterKit, Placeholder, CodeBlockLowlight, Underline, TaskList, TaskItem, Table, Mathematics, WikiLinkExtension
- WikiLinkExtension wiring:
  - `getAllNotes`: return `allLoadedNotes` PLUS pseudo-entries for experiments (with `_entityType: 'experiment'`, `_entityId: exp.id`, `_navigateUrl: /projects/${project.id}/experiments`, `name: exp.name`) and linked literature (with `_entityType: 'paper'|'website'`, `_entityId`, `_navigateUrl: /library/paper/${id} or /library/website/${id}`, `name: title`)
  - Autocomplete dropdown renders type badges: small colored pill showing "Note", "Experiment", "Paper", "Website"
  - `onWikiLinkClick(noteName)`: (1) find matching note in allLoadedNotes -> open in tab, (2) find matching experiment -> `window.open(navigateUrl, '_blank')`, (3) find matching paper/website -> `window.open(navigateUrl, '_blank')`, (4) no match -> auto-create note in current editing scope via `createNote`
- Auto-save: debounced (1s) save on content change via `notesApi.update(noteId, { content })`

**Note creation routing:**
- Track `activeEditorSourceRef` (useRef) — set to the `sourceKey` of the note currently being edited
- `createNote(data)` function: if `activeEditorSourceRef.current` starts with 'experiment:', extract expId and call `notesApi.createForExperiment(expId, data)`. Otherwise call `notesApi.createForProject(project.id, data)`
- New note button in sidebar header and per-folder. Created note opens in a new tab automatically

**Important patterns (from research pitfalls):**
- Do NOT use virtual folder IDs (`exp_` prefix) from old ProjectNotes. Experiments are real ItemFolder components
- Do NOT inject virtual folder nodes into notes arrays
- `activeEditorSourceRef` determines create routing, not folder selection

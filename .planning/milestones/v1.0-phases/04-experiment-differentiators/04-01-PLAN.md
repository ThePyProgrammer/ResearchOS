**Backend service** — Add two functions to `experiment_service.py`:

1. `duplicate_experiment(exp_id: str, deep: bool = False) -> Optional[Experiment]`:
   - Fetch source via `get_experiment(exp_id)`. Return None if not found.
   - Call `create_experiment()` with: `project_id=source.project_id`, `parent_id=source.parent_id`, `rq_id=source.rq_id`, `name=source.name + " (copy)"`, `status="planned"`, `config=source.config`, `metrics={}`.
   - If `deep=True`: fetch all experiments for the project via `list_experiments(source.project_id)`, then call `_deep_clone_children(source.id, new_exp.id, source.project_id, all_exps)`.
   - Return the new root experiment.

2. `_deep_clone_children(source_parent_id: str, new_parent_id: str, project_id: str, all_exps: list) -> None`:
   - Filter `all_exps` for children where `parent_id == source_parent_id`.
   - Sort by `position` for stable ordering.
   - For each child: call `create_experiment()` with `parent_id=new_parent_id`, `name=child.name` (no " (copy)" for children), `status="planned"`, `config=child.config`, `metrics={}`.
   - Recursively call `_deep_clone_children(child.id, cloned.id, project_id, all_exps)` for each child.

**Backend router** — Add to `experiments.py`:

```python
@router.post("/api/experiments/{exp_id}/duplicate", status_code=201)
async def duplicate_experiment_route(exp_id: str, deep: bool = False):
    result = experiment_service.duplicate_experiment(exp_id, deep=deep)
    if result is None:
        raise HTTPException(status_code=404, detail=NOT_FOUND)
    return JSONResponse(result.model_dump(by_alias=True), status_code=201)
```

The `deep` parameter comes as a query param: `?deep=true`.

**Add parentId prop to ExperimentNode.** ExperimentNode currently does not receive its parent's ID. This is needed so that "Duplicate" creates a sibling (same parent), not a child.

**Duplicate handler logic — two paths:**

A. **Leaf experiments (no children):** Open the existing `ExperimentCreateModal` pre-filled. Add `initialName` and `initialConfig` props to `ExperimentCreateModal`:

   - `initialConfig` defaults to `{}`, used to initialize configRows state: `Object.entries(initialConfig).map(([k, v]) => ({ key: k, value: String(v) }))`.
   - When `initialName` or `initialConfig` are provided, pre-populate the form. The parentId passed to the modal should be the current experiment's `parentId` prop (sibling creation), NOT `experiment.id`.

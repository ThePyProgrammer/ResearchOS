  <action>
Enhance the Step 3 (Preview) in CSVImportModal with full interactive editing:

1. **Row exclusion checkboxes:**
   - Add `_excluded: false` to each node in the preview tree state
   - Checkbox on each leaf node — unchecking sets `_excluded: true`
   - Group nodes show a tri-state checkbox: checked (all children included), unchecked (all excluded), indeterminate (mixed)
   - Toggling group checkbox toggles all descendants
   - Excluded nodes render with `opacity-40` and strikethrough name
   - Bottom summary updates: "X of Y experiments selected for import"

2. **Double-click inline rename:**
   - Both group and leaf names support double-click to enter edit mode
   - On double-click: replace name span with an input, auto-focus, select all text
   - On Enter or blur: save the new name to the preview tree state
   - On Escape: cancel edit, restore original name
   - Use the same pattern as ExperimentNode's double-click rename (Phase 3 decision)

3. **Collision resolution per-row:**
   - Leaves with `_collision` set show an amber background (`bg-amber-50 border-l-2 border-amber-400`)
   - Next to the collision badge, show a dropdown: "Create new" (default) / "Update metrics" / "Skip"
   - Store selection as `_collisionAction` on the node
   - When "Update metrics" selected, also store `existing_id` from the collision match
   - Collision detection uses `detectCollision` from csvImportUtils — already computed in buildImportTree
   - For group-level collisions (group name matches existing experiment group under same parent): show "(merges with existing)" label in slate text, auto-set to merge behavior (add children under existing group). Use the existing group's real ID as the parent for its children.

4. **Back navigation state reset:**
   - When user clicks "Back" from Step 3 to Step 2: fully clear the preview tree state
   - Clear all collision overrides, renames, and exclusions
   - The tree will be rebuilt from scratch when "Next" is clicked in Step 2 again
   - This prevents stale tmp_id references per research pitfall 2

5. **Confirm step updates (Step 4):**
   - Filter out `_excluded: true` nodes before BFS flattening
   - Map `_collisionAction` to `collision_action` field in payload
   - Map `_collision.id` to `existing_id` when action is "update"
   - Pass `merge_metrics` from the Step 2 toggle
   - Show counts by action: "Creating X new, updating Y, skipping Z"

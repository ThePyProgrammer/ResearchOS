function detectType(raw) {
  const trimmed = String(raw).trim()
  if (trimmed === 'true') return true
  if (trimmed === 'false') return false
  const num = Number(trimmed)
  if (trimmed !== '' && !isNaN(num)) return num
  return trimmed
}

  <action>
Implement all functions in `csvImportUtils.js` to make the tests pass:

1. **detectType(raw)** — already implemented (copied from ProjectDetail.jsx).

2. **autoDetectColumnRoles(headers, sampleRows)** — For each header, sample all values in that column across sampleRows. If all non-empty values parse as numbers via detectType, assign "metric". Otherwise assign "config". Empty column names get "skip". Return object: `{ [header]: "metric" | "config" | "skip" }`.

3. **autoGenerateName(row, configCols, maxLength = 60)** — Concatenate `key=value` pairs joined by `_` for each configCol. If result exceeds maxLength, truncate individual values (not keys) and append "...". If configCols is empty, return `"experiment_N"` using a counter or row index.

4. **detectCollision(name, parentTmpId, existingExperiments, targetParentId)** — Find an experiment in existingExperiments where `exp.name === name` AND `exp.parentId === resolvedParentId` (where resolvedParentId is targetParentId if parentTmpId is null, otherwise look up the real parent). Return the matching experiment or null.
   - IMPORTANT: Only match within the same parent scope (per user decision). Two experiments named "resnet" under different parents are NOT collisions.

5. **buildImportTree(rows, mapping, existingExps)** — Per research Pattern 2:
   - mapping shape: `{ nameCol, groupCols: [{col, priority}], configCols: [], metricCols: [], skipCols: [] }`
   - groupCols sorted by priority (Group 1 first)
   - Use path-keyed lookup to deduplicate group nodes
   - Group nodes get config containing their group column values
   - Leaf experiments get config containing ALL config columns PLUS all group column values (per user decision CSV-02)
   - Metrics from metricCols via detectType
   - If no nameCol, use autoGenerateName
   - Each node gets: `_tmpId`, `_type` ("group"/"leaf"), `name`, `config`, `metrics`, `children`, `parentTmpId`, `_collision` (for leaves)
   - `_collision` populated via detectCollision

6. **bfsFlattenImportTree(roots)** — Queue-based BFS traversal. Push all roots, then shift and push children. Returns flat array with parents always before children.

7. **mergeMetrics(existing, incoming, merge)** — If merge=true: `{...existing, ...incoming}` (incoming wins conflicts). If merge=false: return `incoming` only.

Run tests after each function implementation to confirm progressive GREEN state.

**Part A: Add exported pure helper functions** (after existing helpers like unionKeys, before ExperimentSection):

1. `buildColumns(flatTree)` — Returns array of column descriptors:
   - 5 fixed columns: `{id:'type_icon', label:'', type:'fixed', width:40, sortable:false}`, `{id:'name', label:'Name', type:'fixed', width:220, sortable:true}`, `{id:'status', label:'Status', type:'fixed', width:110, sortable:true}`, `{id:'parent', label:'Group', type:'fixed', width:140, sortable:true}`, `{id:'created_at', label:'Created', type:'fixed', width:100, sortable:true}`
   - Config columns from `unionKeys(flatTree.map(e => e.config || {}))`: `{id:'config::${key}', label:key, type:'config', width:120, sortable:true, field:'config', key}`
   - Metric columns from `unionKeys(flatTree.map(e => e.metrics || {}))`: `{id:'metric::${key}', label:key, type:'metric', width:120, sortable:true, field:'metrics', key}`

2. `applyFilter(filter, exp)` — Evaluates a single filter against an experiment:
   - Resolves cell value from filter.column: 'status' -> exp.status, 'name' -> exp.name, 'parent' -> exp._parentId, 'config::key' -> exp.config?.[key], 'metric::key' -> exp.metrics?.[key]
   - Operators: 'eq', 'neq', 'gt', 'lt', 'between' (value=[low,high]), 'empty', 'notempty', 'is' (value is array, checks includes), 'isnot', 'contains' (string includes)
   - Returns boolean

3. `sortRows(rows, sort, parentMap)` — Sorts rows by a single column:
   - sort is `{column, direction:'asc'|'desc'}` or null (returns rows unchanged)
   - Uses `getCellValue(column, exp, parentMap)` helper to resolve values (same resolution as applyFilter, plus 'parent' resolves to parent name via parentMap)
   - Nulls/undefined sort last regardless of direction
   - Numbers compare numerically, strings use localeCompare

4. `getCellValue(column, exp, parentMap)` — helper used by sort and table rendering

Export all 4 functions: `export { buildColumns, applyFilter, sortRows, getCellValue }`

Create `ExperimentTableView` component inside ProjectDetail.jsx (before ExperimentSection). Props: `{ flatTree, selectedLeafIds, onToggle, fetchExperiments, projectId, libraryId }`.

**Column derivation:**
```js
const allColumns = useMemo(() => buildColumns(flatTree), [flatTree])
const visibleColumns = allColumns // Plan 02 adds column picker filtering
```

**Sort state:**
```js
const [sort, setSort] = useState(null) // { column, direction }
function handleSort(colId) {
  setSort(prev => {
    if (!prev || prev.column !== colId) return { column: colId, direction: 'asc' }
    if (prev.direction === 'asc') return { column: colId, direction: 'desc' }
    return null // third click clears
  })
}
```

**Derived data:**
```js
const parentMap = useMemo(() => buildParentMap(flatTree), [flatTree])
const sortedRows = useMemo(() => sortRows(flatTree, sort, parentMap), [flatTree, sort, parentMap])
```

**Header color helper:**
```js
function headerBgClass(colType) {
  if (colType === 'config') return 'bg-blue-50/60'
  if (colType === 'metric') return 'bg-emerald-50/60'
  return 'bg-slate-50'
}
```

**Table structure:**
```jsx
<div className="overflow-auto max-h-[calc(100vh-300px)] border border-slate-200 rounded-lg">
  <table className="border-collapse text-sm w-max min-w-full" style={{ tableLayout: 'fixed' }}>
    <thead>
      <tr>
        {/* Checkbox column — sticky top+left, z-30 */}
        <th className="sticky top-0 left-0 z-30 bg-slate-50 border-b border-r border-slate-200 w-10 px-2 py-2">
          {/* Select all checkbox — indeterminate if some selected */}
          <input type="checkbox" ... />
        </th>
        {visibleColumns.map(col => (
          <th key={col.id}
            className={`sticky top-0 z-20 border-b border-slate-200 px-3 py-2 text-left text-xs font-medium text-slate-500 uppercase tracking-wider whitespace-nowrap ${headerBgClass(col.type)} ${col.id === 'name' ? 'sticky left-10 z-30' : ''}`}
            style={{ width: col.width, minWidth: col.width }}
            onClick={() => col.sortable && handleSort(col.id)}
          >
            <div className="flex items-center gap-1 cursor-pointer select-none">
              {col.label}
              {/* Sort indicator */}
              {sort?.column === col.id && (
                <Icon name={sort.direction === 'asc' ? 'arrow_upward' : 'arrow_downward'} className="text-[12px] text-blue-500" />
              )}
            </div>
          </th>
        ))}
      </tr>
    </thead>
    <tbody>
      {sortedRows.map(exp => (
        <tr key={exp.id} className="hover:bg-slate-50/50 border-b border-slate-100 cursor-pointer">
          {/* Checkbox — sticky left, z-10 */}
          <td className="sticky left-0 z-10 bg-white border-r border-slate-100 px-2 py-2">
            <input type="checkbox" checked={selectedLeafIds.has(exp.id)} onChange={() => onToggle(exp)} />
          </td>
          {visibleColumns.map(col => (
            <td key={col.id}
              className={`px-3 py-2 text-xs text-slate-700 whitespace-nowrap ${col.id === 'name' ? 'sticky left-10 z-10 bg-white border-r border-slate-100 font-medium' : ''}`}
              style={{ width: col.width, minWidth: col.width }}
            >
              {renderCellValue(col, exp, parentMap)}
            </td>
          ))}
        </tr>
      ))}
    </tbody>
  </table>
</div>
```

**Cell rendering** (`renderCellValue` function):
- `type_icon`: `<Icon name={exp.children?.length > 0 ? 'folder' : 'science'} className="text-[16px] text-slate-400" />`
- `name`: `<span className="font-medium text-slate-800">{exp.name}</span>`
- `status`: Render status badge using `experimentStatusConfig[exp.status]` — same pill as tree view
- `parent`: Find parent name from parentMap: `flatTree.find(e => e.id === exp._parentId)?.name || '—'`
- `created_at`: Format as date string (e.g., `new Date(exp.created_at).toLocaleDateString()`)
- `config::key`: `exp.config?.[col.key] ?? '—'` (plain text for now; Plan 02 adds inline editing)
- `metric::key`: `exp.metrics?.[col.key] ?? '—'`

**Select-all checkbox logic:**
- Checked if all sortedRows are in selectedLeafIds
- Indeterminate if some but not all are selected
- On change: if all selected, clear all; otherwise select all sortedRows

**Empty state:** If flatTree is empty (handled by ExperimentSection's existing empty state, so this won't render). But if sortedRows is empty after filtering (Plan 03), show a "No matching experiments" row.

Note: Status column click does NOT open side panel yet (Plan 03 handles the detail panel). For now, clicking a row does nothing beyond checkbox selection.

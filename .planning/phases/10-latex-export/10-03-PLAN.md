    **2. Create `frontend/src/utils/latexExport.js`:**

    Install JSZip first: `npm install jszip`

    Export these functions:

    a) `buildFullLatex({ body, template, title, author, baseName, usedKeys })` — assembles complete .tex document:
    - Template preamble from `latexTemplates.js`
    - `\title{title}` and `\author{author}` if provided
    - `\begin{document}`
    - `\maketitle` if title is provided
    - `body` (the serialized LaTeX content)
    - If `usedKeys.size > 0`: `\bibliographystyle{style}` + `\bibliography{baseName}`
    - `\end{document}`

    b) `generateBibContent(citedItems)` — takes array of paper/website objects, mirrors bibtex_service.py logic:
    - Import `makeCitationKey` and `deduplicateKeys` from `citationKeys.js`
    - Run items through `deduplicateKeys` to get `_resolvedKey` for each
    - For each item, generate a BibTeX entry string matching the format in `bibtex_service.py`:
      - Papers: `@article` or `@inproceedings` based on venue keywords
      - Websites: `@misc`
      - Fields: title, author (in "Last, First and Last, First" format), year, journal/booktitle, doi, eprint, url, abstract
      - Use `_resolvedKey` as the citation key (NOT the base key)
    - Return the full .bib string (entries joined by double newline)

    c) `folderToLatex(folder, notes, allNotes)` — for folder export:
    - Filter `allNotes` to get children of `folder` that are files
    - Sort by provided order (or alphabetical as default)
    - For each child note, serialize with `htmlToLatex(note.content)`
    - Use note name as `\section{name}` header
    - Concatenate all sections
    - Collect all `usedKeys` across all notes
    - Return `{ body, usedKeys }`

    d) `async downloadLatexZip({ texContent, bibContent, baseName })`:
    - Import JSZip
    - Create new JSZip instance
    - Add `{baseName}.tex` and `{baseName}.bib` (if bibContent non-empty)
    - Generate blob
    - Create temporary anchor element, trigger download, revoke URL

    Test all behaviors listed. For JSZip tests, verify the function calls JSZip correctly (can use a simple integration test that checks the zip blob is non-null, or mock JSZip).

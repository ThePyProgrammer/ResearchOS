<objective>
Build the CitationExtension tiptap mark that lets users insert citation references via @-mention in the notes editor.

Purpose: This is the authoring-side of citations. Users type @ to open a popup, search for papers/websites, and insert an inline citation chip. The chip persists in the HTML as a `<span>` with data attributes that the serializer (Plan 01) reads to produce `\cite{key}`. The extension follows the exact same `Mark.create` + `Suggestion` architecture as the existing `WikiLinkExtension.js`.

Output: `CitationExtension.js` with full popup, context menu, and tooltip. CSS styles in `index.css`. Unit tests for `extractCitations` in `CitationExtension.test.js`.
</objective>

```javascript
// Factory function pattern:
export function createWikiLinkExtension({ getAllNotes, onWikiLinkClick }) {
  return Mark.create({
    name: 'wikiLink',
    inclusive: false,
    addAttributes() { return { name: { ... }, noteId: { ... } } },
    parseHTML() { return [{ tag: 'span[data-wiki-name]' }] },
    renderHTML({ HTMLAttributes }) {
      return ['span', mergeAttributes({ class: 'wiki-link' }, HTMLAttributes), 0]
    },
    addProseMirrorPlugins() {
      return [
        new Plugin({ props: { handleClick, handleDOMEvents: { contextmenu } } }),
        Suggestion({ editor, char: '[[', allowSpaces: true, items, command, render })
      ]
    },
  })
}
export function extractWikiLinks(htmlContent) { /* regex-based extraction from HTML */ }
```

  <files>frontend/src/components/CitationExtension.js</files>
  <action>
    Create `frontend/src/components/CitationExtension.js` following the `WikiLinkExtension.js` pattern exactly.

    **Factory function signature:**
    ```javascript
    export function createCitationExtension({ getLinkedItems, getAllLibraryItems, onAutoLink, onOpenItem, getBibtexEntry })
    ```
    - `getLinkedItems()` — returns array of project-linked papers + websites (shown first in popup)
    - `getAllLibraryItems()` — returns array of all library papers + websites (shown under "Search full library..." divider)
    - `onAutoLink(paperId, websiteId)` — called when citing a non-linked library item to auto-link it to the project
    - `onOpenItem(paperId, websiteId)` — called from context menu "Open paper" to navigate
    - `getBibtexEntry(paperId, websiteId)` — returns a Promise resolving to a BibTeX string for "Copy BibTeX entry" context menu action. The function body is NOT created here — it is a callback provided by the consumer (Plan 03 wires it to `papersApi.exportBibtex`). This extension only calls it and copies the result to clipboard.

    **Mark definition:**
    - `name: 'citation'`
    - `inclusive: false`
    - Attributes: `paperId` (data-cite-paper-id), `websiteId` (data-cite-website-id), `citationKey` (data-cite-key), `displayLabel` (data-cite-label)
    - `parseHTML`: `[{ tag: 'span[data-cite-key]' }]`
    - `renderHTML`: `['span', mergeAttributes({ class: 'citation-chip' }, HTMLAttributes), 0]`

    **ProseMirror plugins (same two-plugin pattern as WikiLink):**

    Plugin 1 — Click + Context menu handler:
    - Left-click on citation span: open context menu (NOT navigate — different from WikiLink which navigates on click)
    - Right-click on citation span: also open context menu
    - Context menu items:
      1. Header: shows paper title (truncated) + citation key
      2. "Open paper" — calls `onOpenItem(paperId, websiteId)` and opens in new tab
      3. "Copy citation key" — copies `citationKey` to clipboard via `navigator.clipboard.writeText()`
      4. "Copy BibTeX entry" — calls `await getBibtexEntry(paperId, websiteId)`, copies result to clipboard
      5. Separator
      6. "Remove citation" — removes the mark but keeps the text (same as WikiLink's "Remove link")
    - Context menu styling: same CSS-in-JS pattern as WikiLink's `showContextMenu()` with `position:fixed`, white bg, rounded shadow

    Plugin 2 — Suggestion with `char: '@'`:
    - `allowSpaces: true`
    - `items({ query })`: First show `getLinkedItems()` filtered by query. If query is non-empty and results < 5, add a "Search full library..." divider item, then show matching `getAllLibraryItems()` results. Each item has: `{ id, title, authors, year, type: 'paper'|'website', isLinked: bool }`.
    - `command({ editor, range, props })`:
      - Import `makeCitationKey` and `makeCitationLabel` from `../utils/citationKeys.js` at the top of the file
      - Generate key: `makeCitationKey({ authors: props.authors, year: props.year, title: props.title })`
      - Generate label: `makeCitationLabel({ authors: props.authors, year: props.year, title: props.title })`
      - Insert mark with text = label (e.g., "(Vaswani et al., 2017)")
      - If `!props.isLinked`: call `onAutoLink(props.id, props.type === 'website' ? props.id : null)` to auto-link to project
      - Add a trailing space after the mark
    - `render()`: Same imperative DOM popup pattern as WikiLink. Show items with:
      - Paper icon (description) or website icon (language) based on type
      - Title (truncated, bold if linked)
      - Author last name + year in muted text
      - "Linked" badge for project-linked items
      - Divider item "Search full library..." rendered as a section header (not clickable)

    **Hover tooltip (CSS-only or lightweight):**
    - Add a `title` attribute on the rendered span in `renderHTML` that shows: "Title | Authors | Year | Venue | Key: citationKey"
    - This uses the browser's native tooltip. A more elaborate tooltip can be deferred.

    **@ trigger collision avoidance (Pitfall 6):**
    - The Suggestion plugin should NOT fire if the character immediately before @ is a word character (letter/digit). Check using a custom `allow` function or by checking the character at `range.from - 1` in the `items` callback. This prevents triggering on email addresses like `user@domain`.

    **Export `extractCitations(htmlContent)`** function (parallel to `extractWikiLinks`):
    - Parse HTML with regex (same approach as extractWikiLinks — NOT DOMParser, since this is called on stored HTML strings outside the editor)
    - Find all `<span ... data-cite-key="..." ...>` elements
    - Extract `citationKey`, `paperId`, `websiteId` from attributes
    - Deduplicate by citationKey
    - Return array of `{ citationKey, paperId, websiteId }`

    Test cases:


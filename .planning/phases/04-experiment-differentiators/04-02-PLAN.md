**Part A — Selection state in ExperimentSection:**

1. Add `selectedLeafIds` state (Set) in ExperimentSection.
2. Create `collectLeafIds(node)` helper: if node has no children, return `[node.id]`; otherwise `node.children.flatMap(collectLeafIds)`. Reuse the tree structure from `buildExperimentTree()`.
3. Create `handleToggleNode(exp)` handler: get `leaves = collectLeafIds(exp)`. If all leaves are already selected, remove them all; otherwise add them all. This handles both leaf clicks (toggles single) and parent clicks (toggles all descendants).
4. Pass `selectedLeafIds`, `onToggle={handleToggleNode}` props to each top-level ExperimentNode.

**Part B — Checkbox UI in ExperimentNode:**

1. Add `selectedLeafIds` and `onToggle` to ExperimentNode's props.
2. Thread these props to recursive children renders.
3. Render a checkbox before the experiment name. Use Material Symbols Outlined icons:
   - Leaf node: `check_box` (if selected) or `check_box_outline_blank` (if not). On click: `onToggle(experiment)`.
   - Parent node: `check_box` (if ALL leaf descendants selected), `indeterminate_check_box` (if SOME selected), `check_box_outline_blank` (if none). On click: `onToggle(experiment)`.
4. Checkbox styling: `opacity-0 group-hover:opacity-100` on the experiment row (show on hover), but if ANY item is selected (`selectedLeafIds.size > 0`), show all checkboxes always (opacity-100).
5. Checkbox color: `text-blue-600` when checked/indeterminate, `text-slate-400` otherwise.
6. Determine checkbox state for parent nodes: `const leaves = collectLeafIds(experiment); const checkedCount = leaves.filter(id => selectedLeafIds.has(id)).length; const allChecked = checkedCount === leaves.length; const someChecked = checkedCount > 0 && !allChecked;`

**Part C — Floating action bar in ExperimentSection:**

**Part D — CompareModal component:**

Create a `CompareModal` component (can be defined inline in ProjectDetail.jsx or extracted — keep inline if under 200 lines).

Props: `experiments` (array of leaf Experiment objects), `open`, `onClose`.

State:
- `activeTab` ('metrics' | 'config'), default 'metrics'
- `highlightBest` (boolean), default true
- `lowerIsBetter` (object mapping metricKey -> boolean), default {}
- `changedOnly` (boolean), default false

**Metrics tab:**
- Build `metricKeys = [...new Set(experiments.flatMap(e => Object.keys(e.metrics || {})))].sort()`
- Render a table: first column = metric key name (sticky left), subsequent columns = one per experiment (header = experiment name + status badge).
- For each cell: if experiment has that metric key, show the value; if not, show `---` in `text-slate-300 italic`.
- Best-value highlighting (when `highlightBest` is true): for each metric row, find best value using `getBestValue(key, experiments, lowerIsBetter)`. If cell value === best, apply `font-bold text-emerald-700 bg-emerald-50`. Non-numeric values (`typeof v !== 'number'`) get no highlighting.
- Controls above table: toggle "Highlight best" checkbox. Per-metric "lower is better" toggle: add a small down-arrow icon next to metric key name; clicking it toggles `lowerIsBetter[key]`. When toggled, the icon changes to indicate direction (use `arrow_downward` for lower-is-better, `arrow_upward` for higher-is-better).

**Config tab:**
- Build `configKeys = [...new Set(experiments.flatMap(e => Object.keys(e.config || {})))].sort()`
- Same table layout as metrics tab.
- For each cell, classify: if value is undefined/null, show `---` in `text-slate-300 italic` (missing). If all defined values are the same string, no highlight (unchanged). If the value exists but differs from others, apply `bg-amber-50 text-amber-800` (changed). If the key exists in this experiment but is missing in some others, apply `bg-emerald-50 text-emerald-800` (added).
- "Changed only" toggle: when enabled, filter out config rows where ALL experiments have the same value for that key (i.e., `vals.every(v => String(v) === String(vals[0]))`).
- Controls above table: toggle "Changed only" checkbox.

**Modal chrome:**
- Use `WindowModal` with `iconName="compare_arrows"`, `iconWrapClassName="bg-blue-100"`, `iconClassName="text-[16px] text-blue-600"`.
- Title: `Compare {N} Experiments`.
- Tab bar below title with "Metrics" and "Config" tabs.
- Table wrapper: `overflow-x-auto` for horizontal scrolling when many experiments.
- Column widths: metric/config key column `min-w-[160px]`, experiment columns `min-w-[120px]`.

**Wiring in ExperimentSection:**
- Add `compareOpen` state (boolean).
- When floating bar "Compare" is clicked, set `compareOpen = true`.
- Derive `compareExperiments` by filtering `flatExperiments` (the flat list used elsewhere in ExperimentSection) to only those whose ID is in `selectedLeafIds`. Pass this array to CompareModal.
- On CompareModal close, keep selection intact (user may want to re-open).

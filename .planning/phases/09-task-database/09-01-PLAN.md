1. **Migration 021_task_database.sql** — Create three tables following the schema from 09-RESEARCH.md Pattern 1:
   - `task_columns`: id (TEXT PK, e.g. `tcol_{uuid4.hex[:8]}`), project_id (FK projects ON DELETE CASCADE), name, color (default '#94a3b8'), position (INT default 0), created_at
   - `tasks`: id (TEXT PK, e.g. `task_{uuid4.hex[:8]}`), project_id (FK projects ON DELETE CASCADE), column_id (FK task_columns ON DELETE RESTRICT — NOT CASCADE per locked decision), title, description (TEXT nullable), priority (TEXT default 'none'), due_date (TEXT nullable, YYYY-MM-DD), tags (JSONB default '[]'), custom_fields (JSONB default '{}'), position (INT default 0), created_at, updated_at
   - `task_field_defs`: id (TEXT PK, e.g. `tfd_{uuid4.hex[:8]}`), project_id (FK projects ON DELETE CASCADE), name, field_type (TEXT), options (JSONB default '[]'), position (INT default 0), created_at
   - Add indexes on project_id for all three tables

2. **Models (backend/models/task.py)** — Create Pydantic models inheriting CamelModel (from models.base):
   - TaskColumn, TaskColumnCreate (name, color optional), TaskColumnUpdate (all optional)
   - Task, TaskCreate (title required, column_id required, rest optional), TaskUpdate (all optional, use exclude_unset pattern)
   - TaskFieldDef, TaskFieldDefCreate (name, field_type required, options optional), TaskFieldDefUpdate (all optional)
   - Priority is Literal['high', 'medium', 'low', 'none']
   - field_type is Literal['text', 'number', 'date', 'select', 'multi_select']

3. **Service (backend/services/task_service.py)** — Follow experiment_service.py pattern exactly:
   - `list_or_seed_task_columns(project_id)` — returns columns ordered by position; if empty, seeds 4 defaults: Todo (#93c5fd), In Progress (#fbbf24), Review (#a78bfa), Done (#4ade80)
   - `create_task_column(project_id, data)`, `update_task_column(col_id, data)`, `delete_task_column(col_id, move_to_col_id)` — delete must move all tasks to move_to_col_id first, then delete column
   - `list_tasks(project_id)`, `get_task(task_id)`, `create_task(project_id, data)`, `update_task(task_id, data)`, `delete_task(task_id)`
   - `list_task_field_defs(project_id)`, `create_task_field_def(project_id, data)`, `update_task_field_def(def_id, data)`, `delete_task_field_def(def_id)`
   - Use `exclude_unset=True` for updates (not exclude_none). Use uuid4().hex[:8] for ID generation with appropriate prefix.

4. **Router (backend/routers/tasks.py)** — Follow experiments.py dual-scope pattern:
   - Project-scoped: GET/POST `/api/projects/{project_id}/tasks`, GET/POST `/api/projects/{project_id}/task-columns`, GET/POST `/api/projects/{project_id}/task-field-defs`
   - Task-scoped: PATCH/DELETE `/api/tasks/{task_id}`
   - Column-scoped: PATCH `/api/task-columns/{column_id}`, DELETE `/api/task-columns/{column_id}?move_to={move_to_column_id}` (move_to required on delete)
   - Field-def-scoped: PATCH/DELETE `/api/task-field-defs/{def_id}`
   - Return 404 with `{"error": "not_found", "detail": "..."}` on missing resources
   - All responses use `model.model_dump(by_alias=True)`

5. **Wire router in app.py** — Import and include the tasks router (same pattern as experiments router registration).

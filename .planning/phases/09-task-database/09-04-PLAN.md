  <action>
Build `CalendarView` component inside ProjectTasks.jsx.

**Month grid (hand-built, no library per research recommendation):**
- `getMonthGrid(year, month)` pure function: returns 42-element array (6 rows x 7 cols). Each element is either a Date object (for valid days in the month) or null (for padding cells before day 1 and after last day). Month is 0-indexed JS Date month.
- Header row: Sun, Mon, Tue, Wed, Thu, Fri, Sat labels
- Grid cells: each cell shows the day number (top-left, small text), then up to 3 task chips below. If more than 3 tasks on a date, show "+N more" link — clicking it opens a small popover listing all tasks for that day.
- Today's cell gets a subtle highlight (light blue border or bg)
- Padding cells (null) render as empty gray cells

**Task chips in calendar:**
- Small pill/chip with truncated task title, background color = column.color (the task's status column color)
- Overdue tasks: red background instead of column color (overdue = `task.dueDate < today` where `today = new Date().toISOString().split('T')[0]`)
- Click chip → `onSelectTask(task.id)` to open TaskDetailPanel

**Navigation:**
- State: `currentYear`, `currentMonth` (initialized to current date)
- Prev/Next month buttons (chevron_left, chevron_right icons) in header
- "Today" button resets to current month/year
- Month/Year label in center (e.g., "March 2026")

**Unscheduled sidebar (right panel per locked decision):**
- Shows all tasks where `dueDate` is null
- Each task rendered as a draggable card (title + status color dot)
- Width ~200px, scrollable if many unscheduled tasks
- Header: "Unscheduled" with count badge

**Drag-to-date (assign due date):**
- Use `DndContext` from @dnd-kit/core with `id="task-calendar-dnd"`
- Each calendar date cell is a `useDroppable` target with `id=YYYY-MM-DD` string
- Unscheduled sidebar cards use `useDraggable` with `id=task.id`
- Task chips already on the calendar also use `useDraggable` (for rescheduling)
- Do NOT use SortableContext — no sorting within a day cell (per research note)
- `onDragEnd`: extract the target date from `over.id` (it's the YYYY-MM-DD string). Call `tasksApi.update(activeTaskId, { due_date: targetDate })` → optimistic update → refresh
- `DragOverlay`: render a small card with task title during drag

**Empty state:**
- If no tasks exist at all: "No tasks yet. Create one from the Kanban view or use the list view."
- If tasks exist but none have due dates: calendar is empty, all tasks appear in unscheduled sidebar

Export `getMonthGrid` function for testability.

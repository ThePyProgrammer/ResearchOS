Build `TaskListView` component inside ProjectTasks.jsx.

**Table layout:**
- Default columns (always present): Title, Status (colored badge using column.color), Priority (badge: High=red, Medium=yellow, Low=blue, None=gray), Due Date (red text if overdue), Tags (comma-joined chips)
- Custom field columns: one column per task_field_def, rendering the value from task.customFields[fieldDef.id]
- Each row is clickable → `onSelectTask(task.id)` to open TaskDetailPanel
- Inline editing is NOT required in list view — clicking opens the detail panel for editing (per locked decision: "click card -> side panel")

**Column management:**
- "+" button at end of header row (same pattern as experiment table view column management per locked decision)
- Clicking "+" opens a small dropdown/popover: name input, type selector (text/number/date/select/multi_select), options input for select types, "Add" button
- Creating a field calls `taskFieldDefsApi.create(projectId, { name, field_type, options })` → refresh field defs
- Column visibility toggle via ColumnPicker dropdown (gear icon in header area) — same UX as experiment table
- Column drag-reorder for custom columns (using @dnd-kit with id="task-list-column-dnd")

**Sorting:**
- Click column header to sort ascending, click again for descending, click third time to clear
- Sort state: `{ columnId, direction: 'asc'|'desc' }`
- Sort logic: for Status, sort by column.position; for Priority, sort by rank (high=0, medium=1, low=2, none=3); for dates, sort chronologically; for text, sort alphabetically (case-insensitive); for numbers, sort numerically

**Export `applyTaskFilter` and `sortTaskRows` functions** for testability:
- `export function applyTaskFilter(tasks, filters)` — takes task array and filter descriptors, returns filtered array
- `export function sortTaskRows(tasks, sortConfig, columns)` — takes task array, sort config `{ columnId, direction }`, and columns array; returns sorted copy

**Filtering (reuse FilterBar/FilterChip pattern from experiment table view):**
- FilterBar above the table with "+ Add filter" button
- Each filter chip: column selector dropdown, operator dropdown (varies by type: text→contains/equals/empty/notempty, number→equals/gt/lt/between, date→equals/before/after/between, select→equals/in, multi_select→contains), value input
- Apply filters client-side using the exported `applyTaskFilter` function
- Show filter chips with remove (X) button, same visual style as experiment table

**Custom field CRUD in list header:**
- Right-click or hover menu on custom field column headers: Rename, Change type, Delete
- Rename: inline edit in header
- Delete: confirmation, then `taskFieldDefsApi.remove(defId)` — also removes that key from all tasks' custom_fields (or just leave orphaned data, it won't display)

**Delete from list view:**
- Row hover shows a small trash icon at the end → confirmation → `tasksApi.remove(taskId)` → refresh

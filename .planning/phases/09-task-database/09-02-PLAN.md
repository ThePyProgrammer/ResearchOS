  <action>
Build `KanbanView` component inside ProjectTasks.jsx (or as a section within the file, following the ExperimentSection/ExperimentTableView pattern of keeping view components in the same file).

**Layout:**
- Horizontal scrollable container with one column per task_column, ordered by column.position
- Each column: colored header bar (column.color), column name, task count badge, card list, "+ Add task" button at bottom
- Cards show: title (bold), due date (small text, red if overdue — compare YYYY-MM-DD strings: `task.dueDate < new Date().toISOString().split('T')[0]`), first 1-2 custom field values as small chips. NO priority indicator on cards per locked decision.
- Card click calls `onSelectTask(task.id)` to open TaskDetailPanel
- Column min-width ~280px, gap between columns ~12px

**DnD (card movement between columns):**
- Use `DndContext` from @dnd-kit/core with `id="task-card-dnd"` (separate from any column reorder DnD per research pitfall 3)
- Use `closestCorners` collision detection (NOT closestCenter — per research anti-pattern)
- Use `PointerSensor` with `activationConstraint: { distance: 5 }` to distinguish click from drag
- Each column is a `SortableContext` with `verticalListSortingStrategy`, items = task IDs in that column
- Each card uses `useSortable` with `id=task.id` and `data: { columnId: column.id }`
- `DragOverlay` renders a card clone during drag (opacity 0.8, shadow)
- `onDragEnd`: if the active card's columnId differs from the over container's columnId, call `tasksApi.update(activeId, { column_id: overColumnId })` then optimistically update local state. If same column, reorder positions.
- Append at bottom of target column on cross-column drag (per research recommendation)

**Inline task creation (per column):**
- "+ Add task" button at bottom of each column — clicking shows an inline text input
- User types title, presses Enter → call `tasksApi.create(projectId, { title, column_id: column.id })` → refresh tasks → clear input
- Escape or blur with empty text cancels
- Auto-focus the input when it appears

**Column management (inline):**
- **Rename:** Double-click column header name → inline text input → Enter saves via `taskColumnsApi.update(colId, { name })` → refresh columns
- **Add column:** "+" button after the last column → inline input for name → Enter creates via `taskColumnsApi.create(projectId, { name })` → refresh
- **Delete column:** Small trash icon on column header (visible on hover) → if only 1 column, disable/hide the button (minimum 1 column constraint per research). Otherwise, find the adjacent column (prefer left neighbor, fallback right) and call `taskColumnsApi.remove(colId, adjacentColId)` → refresh. Show a brief confirmation tooltip/toast: "Moved N tasks to [adjacent column name]".
- **Reorder columns:** Use a second `DndContext` with `id="task-column-dnd"` wrapping only the column headers. `SortableContext` with `horizontalListSortingStrategy`. On drag end, update positions via `taskColumnsApi.update` for each moved column.
- **Color:** Small color dot/swatch next to column name. Click opens a simple inline color picker (a grid of 8-10 preset color swatches — no need for a full color picker). Selecting a color calls `taskColumnsApi.update(colId, { color })`.

**Export `isOverdue` function** for testability: `export function isOverdue(dueDate) { ... }` — returns true if dueDate string is before today's YYYY-MM-DD.

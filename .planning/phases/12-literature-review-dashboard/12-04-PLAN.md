  <files>frontend/src/components/HeatmapViz.jsx, frontend/src/components/TimelineViz.jsx, frontend/src/pages/ProjectReviewDashboard.jsx</files>
  <action>
    Create `HeatmapViz.jsx` using d3 for SVG rendering.

    **Props:** `{ papers, rowAxis, colAxis, projectId, onPapersRefresh }`

    `onPapersRefresh` is a callback that triggers re-fetching papers in the parent (after keyword extraction updates tags).

    **Implementation:**

    1. Compute matrix: `const { rows, cols, cells } = buildHeatmapMatrix(papers, rowAxis, colAxis)` (import from ProjectReviewDashboard)

    2. State:
       - `hoveredCell` — null or { row, col, count, papers, x, y }
       - `extracting` — boolean, loading state for Extract Keywords button
       - `extractResult` — null or { updated, skipped, total }

    3. SVG rendering via useEffect (triggered by rows, cols, cells):
       - Measure container width via ref
       - Compute dimensions:
         - Left margin: 120px (for row labels)
         - Top margin: 60px (for rotated column labels)
         - Cell width/height: computed from available space and number of rows/cols, min 24px, max 50px
       - Create scales:
         - `x = d3.scaleBand().domain(cols).range([0, cellWidth * cols.length]).padding(0.05)`
         - `y = d3.scaleBand().domain(rows).range([0, cellHeight * rows.length]).padding(0.05)`
         - `color = d3.scaleSequential(d3.interpolateBlues).domain([0, maxCount])` where maxCount = Math.max(...cells.map(c => c.count))
       - Draw cells as `<rect>`:
         - Fill: `cell.count === 0 ? '#f8fafc' : color(cell.count)` (per RESEARCH.md Pattern 4)
         - Stroke: `cell.count === 0 ? '#e2e8f0' : 'none'` (gap highlighting — dashed border on zero cells)
         - StrokeDasharray: `cell.count === 0 ? '3,2' : 'none'`
         - Rx/Ry: 2 for subtle rounding
       - Draw count text centered in each cell (only if count > 0): white text for dark cells (count > maxCount/2), slate-600 for light cells
       - Row axis labels: truncate to 25 chars with ellipsis, font-size 10px, text-anchor end
       - Column axis labels: rotated -45 degrees, truncate to 20 chars, font-size 10px (per Pitfall 4)
       - Mouse events on cells:
         - mouseenter: setHoveredCell with cell data + mouse position
         - mouseleave: setHoveredCell(null)
         - click: if cell.count === 1, navigate to that paper. If > 1, could show a mini list (Claude's discretion — simplest is to navigate to the first paper)

    4. Hover tooltip (same style as other visualizations):
       ```jsx
       {hoveredCell && (
         <div className="absolute z-50 bg-slate-800 text-white text-[11px] rounded-lg px-3 py-2 pointer-events-none shadow-lg"
           style={{ top: hoveredCell.y + 10, left: hoveredCell.x + 10 }}>
           <div className="font-semibold">{hoveredCell.row} x {hoveredCell.col}</div>
           <div className="text-slate-300 mt-0.5">{hoveredCell.count} paper{hoveredCell.count !== 1 ? 's' : ''}</div>
           {hoveredCell.count > 0 && hoveredCell.count <= 3 && (
             <div className="text-slate-400 mt-1">
               {hoveredCell.papers.map(p => <div key={p.id} className="truncate">{p.title}</div>)}
             </div>
           )}
         </div>
       )}
       ```

    5. "Extract Keywords" button — only shown when rowAxis === 'tags' OR colAxis === 'tags':
       - Positioned in a toolbar strip above the heatmap SVG
       - Count untagged papers: `const untaggedCount = papers.filter(p => !p.tags?.length).length`
       - If untaggedCount === 0: show "All papers tagged" (disabled button)
       - Otherwise: button text = `Extract Keywords (${untaggedCount} untagged)` with a sparkle/auto_awesome icon
       - On click:
         - Set extracting = true
         - Call `projectPapersApi.extractKeywords(projectId)` (import from api.js)
         - On success: set extractResult, call `onPapersRefresh()` to re-fetch papers in parent
         - On error: show error via console.error (or simple inline error text)
         - Set extracting = false
       - While extracting: show spinner and "Extracting keywords..." text (per Pitfall 5)

    6. Tags sparsity callout (per Research Open Question 2):
       - When tags axis is selected and > 50% of papers have empty tags, show a yellow callout:
         "Most papers don't have tags yet. Click 'Extract Keywords' to auto-generate tags from paper abstracts."

    7. Empty state: if no papers, show centered message.

    8. Wire into ProjectReviewDashboard.jsx:
       - Import TimelineViz and HeatmapViz
       - Replace timeline placeholder with `<TimelineViz papers={papers} colorBy={timelineOptions.colorBy} projectId={projectId} />`
       - Replace heatmap placeholder with `<HeatmapViz papers={papers} rowAxis={heatmapOptions.rowAxis} colAxis={heatmapOptions.colAxis} projectId={projectId} onPapersRefresh={refreshPapers} />`
       - Add `refreshPapers` function to the parent that re-fetches the paper data and triggers re-render
       - Default heatmap axes: rowAxis='venue', colAxis='year' (NOT tags, since tags are likely empty — per Research Open Question 2)

    Run all tests to verify nothing broken.

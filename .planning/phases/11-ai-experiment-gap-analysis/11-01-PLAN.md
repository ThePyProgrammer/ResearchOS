<objective>
Build the backend for AI experiment gap analysis: Pydantic models for gap suggestions, a pydantic-ai agent that analyzes experiment trees against linked papers to detect missing experiments, and a FastAPI endpoint to trigger analysis.

Purpose: Enables the frontend to call a single POST endpoint that returns structured gap suggestions (missing baselines, ablation gaps, config sweeps, replications) with paper references and suggested configs.

Output: Working POST /api/projects/{id}/gap-analysis endpoint returning GapSuggestion list, backed by pydantic-ai Agent with structured output. Full test coverage.

       - `_make_gap_agent() -> Agent`: creates pydantic-ai Agent with get_pydantic_ai_model("agent"), output_type=GapAnalysisOutput, system_prompt=GAP_ANALYSIS_SYSTEM_PROMPT, defer_model_check=True
       - `async def run_gap_analysis(project_id: str, dismissed_ids: list[str] = []) -> list[GapSuggestion]`:
         a) Fetch experiments via experiment_service.list_experiments(project_id)
         b) Fetch project papers via project_papers_service.list_project_papers(project_id)
         c) For each project paper with paper_id, fetch full paper via paper_service.get_paper()
         d) Build user prompt: serialized tree + serialized papers + "Dismissed suggestion IDs (do not re-suggest similar): {dismissed_ids}"
         e) Call agent.run(user_prompt)
         f) Track cost: RunCostTracker + record_openai_usage
         g) Return result.output.suggestions
       - Log token estimate before calling LLM (len(user_prompt) // 4 as rough token count)

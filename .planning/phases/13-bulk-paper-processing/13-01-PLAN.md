  <action>
    1. Add `extract_keywords_for_items(item_ids: list[str], library_id: str | None = None) -> dict` to `keyword_extraction_service.py`. This function:
       - Iterates item_ids, classifying by prefix: `w_` = website, `gh_` = github_repo, else = paper
       - For each, calls the appropriate get_* service function to load the item
       - Skips items that already have tags (bool(item.tags)) or lack abstract/description text
       - Builds candidates as `[{"id": ..., "title": ..., "abstract": ...}]` — for websites use `description` as abstract, for GitHub repos use `abstract or description`
       - Calls OpenAI with the same `_SYSTEM_PROMPT` and `_MODEL` and `response_format=json_object` as the existing `extract_keywords_for_project`
       - Parses the tag_map response and applies tags via the appropriate update service (PaperUpdate/WebsiteUpdate/GitHubRepoUpdate with tags= field)
       - Records OpenAI usage via `record_openai_usage`
       - Returns `{"updated": N, "skipped": N, "total": N}`
       - If no candidates, returns early with updated=0

    2. Create `backend/services/batch_service.py` with two functions:

       a. `async def batch_index_embeddings(item_ids: list[str]) -> dict`:
          - For each item_id, classify by prefix and call get_paper/get_website/get_github_repo
          - For each found item, call the corresponding `index_paper`/`index_website`/`index_github_repo` from search_service (these are async)
          - Returns `{"processed": len(found_items), "not_found": len(item_ids) - len(found_items)}` since search_service handles skip internally

       b. `def batch_notes_preview(item_ids: list[str]) -> dict`:
          - Queries the notes table for all "AI Notes" folders matching the given item IDs
          - For each item_id, classify by prefix: `w_` = website (check website_id), `gh_` = github_repo (check github_repo_id), else = paper (check paper_id)
          - Query: `get_client().table("notes").select("paper_id, website_id, github_repo_id").eq("name", "AI Notes").eq("type", "folder").execute()`
          - Build a set of item IDs that already have AI Notes folders
          - Returns `{"skip_ids": [...], "process_ids": [...]}` where skip_ids are items with existing AI Notes and process_ids are items without
          - Per user decision (13-CONTEXT.md): "Skip items with existing AI notes — don't overwrite"

    3. Write tests in `backend/tests/test_batch.py`:
